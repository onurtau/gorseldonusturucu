### 📝 2. TARGETSIZE ALGORITHM - BINARY SEARCH OPTİMİZASYONU

#### Problem
Hedef dosya boyutu (targetSize) özelliği:
- **CMYK dosyalarda:** Hedefe ulaşamıyor (çok küçük kalıyor)
- **RGB dosyalarda:** Hedefe ulaşamıyor (çok büyük kalıyor)
- **Linear quality reduction:** -10 sabit adım yetersiz

#### Eski Algoritma (Kaldırıldı)
```javascript
// Basit iterasyon - etkisiz
do {
  currentQuality = Math.max(10, currentQuality - 10);
} while (outputBuffer.length > targetBytes);
```

#### Yeni Algoritma: Binary Search

**Dosya:** `electron/main.js`

**A) Akıllı Kalite Aralığı Belirleme**
```javascript
// Dosya boyutu / hedef boyut oranına göre
const sizeRatio = originalFileSize / targetBytes;

if (sizeRatio > 10) {
  // Çok büyük dosya → düşük kalite
  minQuality = 5;
  maxQuality = 40;
} else if (sizeRatio > 5) {
  // Büyük dosya → orta kalite
  minQuality = 10;
  maxQuality = 60;
} else if (sizeRatio > 2) {
  // Orta dosya → yüksek kalite
  minQuality = 30;
  maxQuality = 80;
} else {
  // Hedefe yakın → çok yüksek kalite
  minQuality = 50;
  maxQuality = 95;
}
```

**B) Binary Search İteratörü**
```javascript
while (iterations < maxIterations && (maxQuality - minQuality) > 1) {
  const currentQuality = Math.round((minQuality + maxQuality) / 2);
  
  // İşle
  outputBuffer = await processImage(currentQuality);
  
  // En iyi sonucu kaydet
  if (diff < bestDiff) {
    bestBuffer = outputBuffer;
    bestQuality = currentQuality;
  }
  
  // Tolerans kontrolü (%5)
  if (diff <= tolerance) break;
  
  // Binary search: aralığı daralt
  if (currentSize > targetBytes) {
    maxQuality = currentQuality - 1; // Kaliteyi düşür
  } else {
    minQuality = currentQuality + 1; // Kaliteyi artır
  }
}
```

**Parametreler:**
- Tolerans: %10 (targetBytes ± 10%)
- Max iterasyon: 10 (JPEG/PNG/WebP/AVIF), 15 (TIFF)

**Sonuç:** ✅ Hedef boyuta %5-10 toleransla ulaşıyor

### 📝 1. WORKER THREAD ARCHİTECTURE

#### A) Worker Pool Manager

**Dosya:** `electron/main.js`

**Yeni Sınıf: WorkerPool**
```javascript
class WorkerPool {
  constructor(workerPath, poolSize = 4) {
    this.workerPath = workerPath;
    this.poolSize = poolSize;
    this.workers = [];      // Worker instance'ları
    this.queue = [];        // Bekleyen task'lar
    this.activeWorkers = new Set(); // Aktif worker ID'leri
  }
  
  async executeTask(task) {
    // Task'ı available worker'a ata veya queue'ya ekle
  }
  
  runOnWorker(worker, task) {
    // Worker'da task çalıştır
    // Progress events forward et
    // Log messages handle et
  }
}
```

**Özellikler:**
- ✅ 4 paralel worker (CPU core sayısına göre)
- ✅ Task queue sistemi (FIFO)
- ✅ Dynamic worker assignment
- ✅ Graceful shutdown handling
- ✅ Event forwarding (progress, log, error)

**Lifecycle:**
```javascript
// Uygulama başlarken
app.whenReady().then(() => {
  createWindow();
  initializeWorkerPool(); // Pool başlat
});

// Uygulama kapanırken
app.on('window-all-closed', () => {
  if (workerPool) {
    workerPool.terminate(); // Tüm worker'ları temizle
  }
  app.quit();
});
```
