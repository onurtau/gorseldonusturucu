## 📊 TEKNİK DETAYLAR

### Content Security Policy (CSP) Nedir?

**Tanım:**
CSP, modern web tarayıcılarının XSS (Cross-Site Scripting) ve veri enjeksiyon saldırılarına karşı koruma sağlayan güvenlik mekanizmasıdır.

**connect-src Directive:**
- `fetch()`, `XMLHttpRequest`, `WebSocket` gibi ağ isteklerini kontrol eder
- Whitelist mantığıyla çalışır (sadece izin verilen URL'lere istek gönderilir)
- Tanımlı olmayan bir URL'e istek gönderilmeye çalışılırsa tarayıcı engeller

**Projemizdeki CSP Ayarları:**

| Directive | İzin Verilen Kaynaklar | Açıklama |
|-----------|------------------------|----------|
| `default-src` | `'self'` | Varsayılan olarak sadece kendi domain |
| `script-src` | `'self'` `'unsafe-inline'` `'unsafe-eval'` | JavaScript çalıştırma |
| `style-src` | `'self'` `'unsafe-inline'` | CSS yükleme |
| `img-src` | `'self'` `data:` `blob:` `https://*.supabase.co` | Görsel kaynakları |
| `font-src` | `'self'` `data:` | Font dosyaları |
| `connect-src` | `'self'` `http://localhost:5000` `https://zylorpix-backend.onrender.com` `https://*.supabase.co` `wss://*.supabase.co` | API istekleri |

**Öğrenilen Ders:**
Environment variables ve CORS ayarları doğru olsa bile, CSP politikası backend URL'ini içermiyorsa tarayıcı istek göndermeden önce engelleyecektir.
