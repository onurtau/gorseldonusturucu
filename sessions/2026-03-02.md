## 🎓 Öğrenilenler

### React Performance Best Practices:

1. **React.memo:**
   - Props değişmediği sürece component re-render olmaz
   - Expensive render'ları olan componentler için şart
   - displayName eklemeyi unutma (DevTools'da görünsün)

2. **useCallback vs useMemo:**
   - `useCallback`: Function'ları memoize et
   - `useMemo`: Hesaplama sonuçlarını memoize et
   - Dependency array doğru olmalı, yoksa stale closure

3. **Component Design:**
   - Helper function'ları component dışına taşı (pure functions)
   - Inline function'lar yerine useCallback kullan
   - map içinde inline onClick yerine memoized handler

4. **Debugging:**
   - React DevTools Profiler ile re-render'ları takip et
   - Why did this render? kontrolü yap
   - displayName ekle, debugging kolaylaşsın

**2. electron/main.js - Cache + Worker Routing:**

```javascript
const { exec } = require('child_process');
const util = require('util');
const execPromise = util.promisify(exec);

// Thumbnail cache (50 entries, FIFO eviction)
const thumbnailCache = new Map();
const MAX_CACHE_SIZE = 50;

ipcMain.handle('image:getPreviewData', async (event, filePath) => {
  // Cache kontrolü
  const stats = await fs.stat(filePath);
  const cacheKey = `${filePath}-${stats.mtime.getTime()}`;
  
  if (thumbnailCache.has(cacheKey)) {
    return thumbnailCache.get(cacheKey);
  }
  
  // Worker Thread'e gönder
  const worker = workerPool[currentWorkerIndex];
  currentWorkerIndex = (currentWorkerIndex + 1) % workerPool.length;
  
  return new Promise((resolve, reject) => {
    const taskId = Date.now() + Math.random();
    
    worker.once('message', (result) => {
      const previewData = `data:image/jpeg;base64,${result.thumbnail}`;
      
      // Cache'e ekle
      if (thumbnailCache.size >= MAX_CACHE_SIZE) {
        const firstKey = thumbnailCache.keys().next().value;
        thumbnailCache.delete(firstKey); // FIFO eviction
      }
      thumbnailCache.set(cacheKey, previewData);
      
      resolve(previewData);
    });
    
    worker.postMessage({
      type: 'generate-thumbnail',
      filePath,
      id: taskId
    });
  });
});
```

### ✅ Session 3.6: ImageMagick Async Çözümü (14:00-16:30)

**Çözüm:** `execSync` → `execPromise` (async/await)

**electron/main.js - convertWithImageMagick Async:**

```javascript
// ÖNCESI (SYNC):
const { execSync } = require('child_process');

async function convertWithImageMagick(inputPath, outputPath, iccProfilePath) {
  const command = `magick "${inputPath}" -profile "${iccProfilePath}" "${outputPath}"`;
  execSync(command); // ⚠️ Main thread bloke!
}

// SONRASI (ASYNC):
const { exec } = require('child_process');
const util = require('util');
const execPromise = util.promisify(exec);

async function convertWithImageMagick(inputPath, outputPath, iccProfilePath) {
  const command = `magick "${inputPath}" -profile "${iccProfilePath}" "${outputPath}"`;
  await execPromise(command); // ✅ Non-blocking!
}
```

2. ✅ `util.promisify()` ile exec'i Promise'e dönüştürdük
3. ✅ `execSync(command)` → `await execPromise(command)`
4. ✅ Function zaten `async`, mevcut await'ler duruyor

## 🎓 Öğrenilenler ve Best Practices

### 1. Main Thread Kuralları

**Asla main thread'de yapma:**
- ❌ Büyük dosya okuma (FileReader.readAsDataURL for 18MB)
- ❌ CPU-intensive computation (image processing)
- ❌ Synchronous external process (execSync)
- ❌ Büyük string/object kopyalama (24MB Base64)

**Main thread'de yapılabilir:**
- ✅ IPC communication
- ✅ UI updates (React setState)
- ✅ Event handling (onClick, onChange)
- ✅ Küçük hesaplamalar (< 16ms)

### 2. Worker Thread Pattern

**Ne zaman kullan:**
- CPU-bound işlemler (image resize, format convert)
- Büyük data transformations
- Parallel işlem yapılabilecek tasklar

**Worker Pool Avantajları:**
- Round-robin distribution (load balancing)
- CPU core'larını etkili kullanma
- Main thread her zaman responsive

### 3. Cache Stratejileri

**Effective Cache:**
- Modification time ile invalidation
- FIFO eviction (memory control)
- Reasonable limit (50 entries)
- Cache key unique ve stable

### 4. Lazy Loading

**Batch İşlemler İçin:**
- Sadece görünen/gerekli data yükle
- 10 dosya → 1 preview = 90% azalma
- First-item preview yeterli (UX açısından)

### 5. Async External Processes

**Node.js Child Process:**
- `execSync` → blocking (NEVER in Electron main)
- `exec` + promisify → non-blocking (ALWAYS use)
- `spawn` → streams için (large output)
